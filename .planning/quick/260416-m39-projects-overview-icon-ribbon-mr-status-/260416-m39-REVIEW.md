## Warnings

### WR-01: Width off-by-one when `mr_info` is None

**File:** `src/joy/widgets/project_list.py:155,174`

**Issue:** `fixed_right` is always computed as `mr_plain_len + 1 + 6` (the constant `1` represents "one space separator between MR strip and ribbon"). When `mr_info is None`, `mr_plain_len` is 0, so `fixed_right = 0 + 1 + 6 = 7`. That space is then emitted at line 174 (`t.append(" ")`). So far so good — the space and budget agree.

However, when `mr_info is not None`, `mr_strip` already ends with a trailing space (line 135: `mr_strip.append(" ")`). That trailing space is included in `mr_plain_len`, so `fixed_right` counts it. But then the formula adds another `+1` (the "1 space between mr and ribbon" comment). This double-counts one character: the actual rendered right section is `mr_strip` (including its trailing space) + ribbon, not `mr_strip + " " + ribbon`. The +1 constant is only correct when no MR strip is present.

Effect: when an MR is shown, `name_budget` is 1 character narrower than intended, causing the project name to truncate 1 character too early.

**Fix:**
```python
# Compute the space separator only when there is no mr_strip
space_between = 0 if mr_info is not None else 1
fixed_right = mr_plain_len + space_between + len(ribbon_icons)
```
Then at line 174 keep the `else: t.append(" ")` branch (which provides the separator when no MR strip).

### WR-02: `pick_best_mr` priority-1 path ignores the `project.repo` guard

**File:** `src/joy/widgets/project_list.py:219-222`

**Issue:** Priority 1 iterates `rel_index.worktrees_for(project)` and returns the first MR whose key matches `(wt.repo_name, wt.branch)`. This bypass means the early return at line 214 (`if project.repo is None: return None`) is the only guard. If `rel_index.worktrees_for()` returns worktrees from a different repo (which it should not, but is untested), an MR from a completely unrelated repo could appear in the row. More practically: the lookup `mr_data.get((wt.repo_name, wt.branch))` is fine, but there is no check that `wt.repo_name == project.repo` before returning. A future call-site change or an edge-case in `worktrees_for` could produce a silently wrong MR badge.

**Fix:**
```python
for wt in rel_index.worktrees_for(project):
    if wt.repo_name != project.repo:
        continue  # guard against cross-repo worktree matches
    mr = mr_data.get((wt.repo_name, wt.branch))
    if mr is not None:
        return mr
```

### IN-02: `status cycle` silently accepts unknown status strings

**File:** `src/joy/widgets/project_list.py:728-729`

**Issue:** `action_toggle_status` uses `cycle.get(project.status, "idle")` — the default `"idle"` silently resets any unrecognised value (e.g., a typo in the TOML, or a future status value added to the model but not yet in the cycle dict). While this is benign in practice, it means a status like `"blocked"` written manually to TOML would silently become `"idle"` on first `g` press with no warning to the user.

This is not a crash, but could confuse users who extend the status set manually.

**Fix (optional):** Add an assertion or warning log for unknown status values, or document that manual TOML values outside `idle/prio/hold` are unsupported:
```python
cycle = {"idle": "prio", "prio": "hold", "hold": "idle"}
if project.status not in cycle:
    # Unrecognised status — reset to idle before cycling
    project.status = "idle"
else:
    project.status = cycle[project.status]
```
