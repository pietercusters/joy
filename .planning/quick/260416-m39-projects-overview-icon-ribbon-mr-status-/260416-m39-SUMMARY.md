## What Was Built

### Task 1 — icons.py + Project.status

- Created `src/joy/widgets/icons.py` with 14 Nerd Font icon constants (MR/CI icons migrated from `worktree_pane.py`, plus ribbon icons from `object_row.py`).
- Updated `worktree_pane.py` to import all ICON_* from `icons.py` — no local definitions remain.
- Added `status: str = "idle"` field to `Project` dataclass. `to_dict()` always serializes it. `_toml_to_projects()` and `_toml_to_archived()` read it with `.get("status", "idle")` for backward compat.

### Task 2 — ProjectRow ribbon + g binding + spacers

- Rewrote `ProjectRow` with static `build_content()` producing Rich.Text:
  - Left: status dot (`●` green for prio, `●` dim for hold, `○` dim for idle)
  - Center: project name (truncated to budget)
  - Right of name: optional MR strip (`!N ` + MR icon + CI icon) when MR data present
  - Far right: 6-icon ribbon (branch ticket thread note terminal worktree) — cyan when present, grey50 dim when absent
- Added `pick_best_mr()` function: prioritizes worktree-linked branch MR, then highest open MR, then highest any MR for the project's repo.
- Added `g` binding → `action_toggle_status()`: cycles `idle → prio → hold → idle`, calls `_save_projects_bg()`, re-renders the single row.
- Added section spacers (`Static("", classes="section-spacer")`) between repo groups in `_rebuild()`.
- Added `_get_available_width()` for row budget (width − 2, floored at 20).
- Updated `update_badges()` to accept `mr_data` dict and call `pick_best_mr()` per row.
- Updated `app._update_badges()` to pass `self._current_mr_data`.
- Removed old `ICON_BRANCH` import from `worktree_pane` and `ICON_CLAUDE` import from `terminal_pane` in project_list.py.

### Task 3 — Human Verify (checkpoint)

This task is a `checkpoint:human-verify` and requires manual verification:
1. Run `joy` — project list should show status dots (○) and icon ribbon on every row.
2. Press `g` on a project — dot should change: ○ (idle) → green ● (prio) → dim ● (hold) → ○ (idle).
3. Quit and reopen joy — status should persist (saved to projects.toml).
4. If a project has MR data loaded (after refresh `r`), its row should show MR strip: e.g. `!123 ` + MR icon + CI icon.
5. Check icons are colored: present objects = cyan, absent = dim grey.
6. Check that repo groups are separated by a blank spacer line.
7. Run `python -m pytest tests/ -q --ignore=tests/tui` — all tests pass.
