# Quick Task 260416-m39: Projects overview icon ribbon, MR status icons, project status toggle (prio/hold/idle) - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Task Boundary

Enrich the project list rows with:
1. A right-aligned icon ribbon (always visible, grey=absent/blue=present): branch, ticket, thread, note, terminal, worktree
2. An MR status strip (only when MR exists for the project): MR state icon + CI pipeline icon + branch status icon — same icons and order as WorktreePane
3. A project status dot (prio=green/hold=dim dot/idle=circle), toggled with `g`
4. Blank line (spacer) between repo sections in the project list

</domain>

<decisions>
## Implementation Decisions

### Icon row layout
- Use Rich.Text with padding — name left-aligned, icons right-aligned, name truncates with … when narrow
- Same pattern as WorktreeRow (single DOM node, no extra containers)
- Content width derived from widget's content_region.width at render time

### MR data source
- Use live API data only: `_current_mr_data` (keyed by `(repo_name, branch)`)
- Active branch determined via `rel_index`: find worktrees linked to the project, pick the active worktree's branch
- Fallback chain: worktree-linked branch MR → highest-numbered open MR for the project's repo → latest closed MR
- MR strip only shown if at least one MR is found; reuses ICON_MR_OPEN / ICON_MR_DRAFT / ICON_CI_* from worktree_pane (centralize to a shared module or import from worktree_pane)

### Project status persistence
- Add `status: str = "idle"` field to `Project` dataclass (default "idle")
- Persisted to TOML under each project's table
- Three states: "prio" (green filled dot), "hold" (dim filled dot), "idle" (empty circle)
- Cycle order: idle → prio → hold → idle
- Keyboard shortcut: `g` in ProjectList (new BINDING)

### White space between sections
- Add a blank spacer row between repo sections in the project list (same `.section-spacer` pattern as WorktreePane)

### Claude's Discretion
- Icon constants (ICON_MR_OPEN, ICON_CI_*, ICON_BRANCH, etc.) should be centralized or imported from worktree_pane to avoid duplication — implementation detail
- Exact icon characters for ticket, thread, note, terminal, worktree (reference PRESET_ICONS or existing constants)
- MR closed/merged icon choice (distinct from open/draft)
- How MR data is passed from app to ProjectList (likely via a new `update_mr_data()` method alongside existing `update_badges()`)

</decisions>

<specifics>
## Specific Ideas

- The status dot/circle: use Unicode block characters — "●" (U+25CF, filled circle) for prio/hold, "○" (U+25CB, empty circle) for idle
- prio: green ●, hold: dim ●, idle: dim ○
- **Status dot is the very first element on the row, before the project name** (confirmed by user)
- Row layout: [status-dot] [project-name...padding...] [MR-strip (if any)] [icon-ribbon]
- Icon ribbon order (left to right): branch, ticket, thread, note, terminal, worktree
- The MR strip sits to the LEFT of the always-present icon ribbon (per user spec)
- `g` binding must not conflict: currently no `g` binding in ProjectList BINDINGS

</specifics>
