<objective>
Fix cursor reset bug: ProjectList and ProjectDetail reset cursor to 0 on every refresh/rebuild
instead of preserving the user's current selection.

Purpose: When background refresh cycles trigger pane rebuilds, the user loses their place
in the list. Both TerminalPane and WorktreePane already implement the correct save/restore
pattern -- this bug applies the same pattern to the two remaining panes.

Output: Both panes preserve cursor identity across rebuilds; tests confirm the behavior.
</objective>

TerminalPane._rebuild_sessions pattern:
```python
# Before DOM rebuild:
saved_name: str | None = None
saved_index = self._cursor
if 0 <= self._cursor < len(self._rows):
    saved_name = self._rows[self._cursor].session_name

# After building new_rows:
if saved_name is not None and new_rows:
    for i, row in enumerate(new_rows):
        if row.session_name == saved_name:
            self._cursor = i
            break
    else:
        self._cursor = min(saved_index, len(new_rows) - 1)
elif new_rows:
    self._cursor = 0
else:
    self._cursor = -1
```
