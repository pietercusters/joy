**Step 2 — Add `_open_worktree_path` immediately after `action_open_ide`:**

```python
@work(thread=True, exit_on_error=False)
def _open_worktree_path(self, path: str) -> None:
    """Open a worktree directory in the configured IDE (background thread)."""
    import subprocess  # noqa: PLC0415
    from pathlib import Path  # noqa: PLC0415
    if not Path(path).exists():
        self.app.call_from_thread(
            self.notify, f"Worktree path not found: {path}", severity="warning", markup=False
        )
        return
    ide = self._config.ide or "Cursor"
    try:
        subprocess.run(["open", "-a", ide, path], check=False)
    except Exception as exc:
        self.app.call_from_thread(
            self.notify, f"Failed to open IDE: {exc}", severity="error", markup=False
        )
```

Note: `@work(thread=True)` methods are run in a thread pool — `self.notify` must be called via `self.app.call_from_thread(self.notify, ...)` inside the worker to avoid calling it from outside the event loop. Check how `_copy_branch` and `_do_open_global` call `self.notify` — if they call it directly (not via call_from_thread), match that pattern for consistency. In Textual `@work(thread=True)`, `self.app.call_from_thread` is the safe pattern; however if existing workers call `self.notify` directly, do the same to stay consistent with the codebase convention.

```python
def set_linked_paths(
    self,
    linked_paths: set[str],
    linked_branches: set[tuple[str, str]],
) -> None:
    """Mark rows as linked or unlinked based on RelationshipIndex data.

    Called from app._update_worktree_link_status after rel_index is computed.
    Rows not found in either linked_paths or linked_branches get the
    .--unlinked CSS class; others have it removed.
    """
    for row in self._rows:
        is_linked = (
            row.path in linked_paths
            or (row.repo_name, row.branch) in linked_branches
        )
        if is_linked:
            row.remove_class("--unlinked")
        else:
            row.add_class("--unlinked")
```

**Step 3 — Add CSS rule for `--unlinked` rows** in `WorktreePane.DEFAULT_CSS`:

Add after the existing `.--highlight` rules:

```css
WorktreeRow.--unlinked {
    color: $text-muted;
    text-style: dim;

    color: $text-muted;

```

The CSS class approach (adding/removing `--unlinked`) is the correct Textual pattern — it applies the visual dim styling without rebuilding the DOM. Note: the label text "(unlinked)" needs to appear in the rendered content. Two options:
- Option A (simple): The dim class already makes the row visually distinct enough — no label text needed
- Option B (explicit): Modify `WorktreeRow.build_content` to accept `is_linked: bool` param and append " (unlinked)" text in dim style when False

**Use Option A** (CSS-only dim) since `set_linked_paths` runs after row creation and rebuilding content would require a full update call. The `--unlinked` CSS class providing dim styling is sufficient to distinguish unlinked worktrees visually, consistent with other dim indicators in the pane (default branch rows, upstream missing indicators).
