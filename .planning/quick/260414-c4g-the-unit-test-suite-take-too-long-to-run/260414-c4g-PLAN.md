<objective>
Mark slow TUI and filter tests with `@pytest.mark.slow` and configure pytest to skip
them by default so `uv run pytest` runs fast (~10s instead of ~264s).

Purpose: Developer feedback loop is blocked by a 264s test suite. Slow tests
(test_tui.py ~150s, test_filter.py ~45s) use Textual pilot which has unavoidable async
overhead. They must remain runnable on demand but should not block fast iteration.

Output: `uv run pytest` skips slow tests. `uv run pytest -m slow` runs only slow tests.
`uv run pytest -m 'slow or not slow'` runs everything.
</objective>
