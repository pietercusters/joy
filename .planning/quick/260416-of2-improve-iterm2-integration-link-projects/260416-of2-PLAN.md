<objective>
Refactor iTerm2 integration to link projects to iTerm2 tabs via tab_id, group the terminal pane by project tab, and remove obsolete name-based matching.

Purpose: Replace fragile session-name-based terminal matching with tab-level project links. Each project gets a dedicated iTerm2 tab identified by tab_id (runtime UUID). The terminal pane shows sessions grouped under their project's tab header instead of Claude/Other. Stale tab IDs (e.g., after iTerm2 restart) are silently auto-healed.

Output: Modified models, store, terminal_sessions, resolver, terminal_pane, app, operations modules + updated tests.
</objective>
