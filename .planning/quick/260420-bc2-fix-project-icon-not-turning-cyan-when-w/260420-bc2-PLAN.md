**Root cause:** `_compute_has()` only checks project.objects for stored WORKTREE/TERMINALS items. The live wt_count and agent_count from the RelationshipIndex are passed to `set_counts()` but never used to influence icon coloring.

**Fix in `ProjectRow`:**

1. Add `wt_count: int = 0` and `agent_count: int = 0` parameters to `build_content()` signature (after `has`).

2. Inside `build_content()`, before building the ribbon, override the `has` dict entries for worktree and terminal using the live counts:

```python
# Live connections override: active worktrees/terminals turn cyan even if not in stored objects
effective_has = dict(has)
if wt_count > 0:
    effective_has["worktree"] = True
if agent_count > 0:
    effective_has["terminal"] = True
```

Then use `effective_has` in the ribbon_icons list instead of `has`.

3. Update `set_counts()` to pass `wt_count` and `agent_count` into `build_content()`:

```python
def set_counts(self, wt_count: int, agent_count: int, mr_info=None, avail_width=None):
    self._mr_info = mr_info
    self._wt_count = wt_count
    self._agent_count = agent_count
    ...
    self.update(self.build_content(
        self.project, self._avail_width, mr_info, self._has,
        wt_count=wt_count, agent_count=agent_count,
    ))
```

4. Store `_wt_count` and `_agent_count` on the instance (initialized to 0 in `__init__`) so that re-renders from other paths (e.g., `action_toggle_status`) also pass the last-known counts. Update `action_toggle_status` in `ProjectList` to pass them.

5. The initial render in `__init__` passes `wt_count=0, agent_count=0` (no live data yet at construction time — badges arrive via `update_badges` after first refresh cycle).

Inspect `text._spans` or render to string and check style names via `Text` internals.

Simpler approach: call `build_content` with different `wt_count`/`agent_count` values and use `text.plain` to confirm the icon characters appear (correctness of which icons), and use `str(text)` or check `text._spans` to verify "cyan" vs "grey50 dim" appears in the appropriate span.

```python
def _spans_for_icon(text: Text, icon_char: str) -> list:

            if icon_char in text.plain[span.start:span.end]]
```
