# Requirements: joy

**Defined:** 2026-04-14
**Core Value:** Every artifact for the active project, openable instantly from one keyboard-driven interface.

## v1.2 Requirements

Requirements for the Cross-Pane Intelligence milestone. Each maps to roadmap phases.

### Foundation

- [ ] **FOUND-01**: Relationship resolver computes Project ↔ Worktree matches (by worktree path or branch name within same repo)
- [ ] **FOUND-02**: Relationship resolver computes Project ↔ Agent matches (by iTerm2 session name matching agents object value)
- [ ] **FOUND-03**: WorktreePane cursor is preserved by identity (repo+branch) across DOM rebuilds triggered by refresh
- [ ] **FOUND-04**: TerminalPane cursor is preserved by identity (session name) across DOM rebuilds triggered by refresh

### Sync

- [ ] **SYNC-01**: Selecting a project updates WorktreePane cursor to first related worktree (keeps current if no match)
- [ ] **SYNC-02**: Selecting a project updates TerminalPane cursor to first related agent session (keeps current if no match)
- [ ] **SYNC-03**: Selecting a worktree updates ProjectList cursor to its related project (keeps current if no match)
- [ ] **SYNC-04**: Selecting a worktree updates TerminalPane cursor to first related agent session (keeps current if no match)
- [ ] **SYNC-05**: Selecting an agent session updates ProjectList cursor to its related project (keeps current if no match)
- [ ] **SYNC-06**: Selecting an agent session updates WorktreePane cursor to first related worktree (keeps current if no match)
- [ ] **SYNC-07**: Focus remains on the active pane during all sync operations
- [ ] **SYNC-08**: User can toggle cross-pane sync on/off via keyboard shortcut
- [ ] **SYNC-09**: Sync toggle state is visible in the footer key hints

### Badge

- [ ] **BADGE-01**: Project rows display count of active related worktrees
- [ ] **BADGE-02**: Project rows display count of active related agent sessions
- [ ] **BADGE-03**: Badge counts update after each background refresh cycle

### Propagation

- [ ] **PROP-01**: When a worktree disappears from git for 2+ consecutive refreshes, its object is auto-removed from the project
- [ ] **PROP-02**: When an MR is detected for a project's branch and no MR object exists yet, it is auto-added silently
- [ ] **PROP-03**: When a worktree's current branch matches a different project's branch object (same repo), the worktree object moves to that project silently
- [ ] **PROP-04**: When an agent session is absent from iTerm2, its project object is marked stale (visually dimmed)
- [ ] **PROP-05**: When a stale agent's session name reappears in iTerm2, the stale marker is cleared
- [ ] **PROP-06**: Branch objects on any project are never modified by auto-propagation
- [ ] **PROP-07**: MR objects are auto-added but never auto-removed by propagation
- [ ] **PROP-08**: Projects without a registered repo are excluded from all propagation

## Future Requirements (v1.3+)

### Post-Propagation Safety

- **PROP-09**: Auto-remove MR objects when PR closes — ambiguous semantics, defer
- **PROP-10**: Undo for auto-mutations — safety net for power users, defer

### Polish

- **SYNC-10**: Sync toggle state persists across restarts — ephemeral is fine for v1.2
- **PERF-01**: Real-time file watching on ~/.joy/ TOML — 30s refresh is sufficient for v1.2

## Out of Scope

| Feature | Reason |
|---------|--------|
| Dim non-matching rows | User explicitly excluded; spatial memory preserved |
| Open worktree → auto-focus agent session | User explicitly excluded |
| Jump-to-project `g` key | Escape already navigates back |
| Auto-create project from discovered worktree | Previously dropped (FLOW-03) |
| Cross-platform support | macOS only; uses iTerm2, app URL schemes, AppleScript |
| MR auto-remove on PR close | Ambiguous; auto-add only |
