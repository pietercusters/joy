## Overview

v1.3 delivered two things: a formal bug-fix phase (Phase 17) hardening iTerm2 integration after the v1.2 quick-task sprint, and a sequence of 21 quick tasks that collectively implemented the Unified Object View, a consistent keystroke dispatch table, cross-pane sync polish, the icon ribbon, project archive/unarchive, the new-project modal extension, and more. ## Phases

### Phase 17: Fix iTerm2 Integration Bugs

**Goal:** Remove automatic iTerm2 tab creation (tabs only via h-key), close entire tab on project delete/archive, isolate all tests from real ~/.joy/ paths
**Depends on:** Phase 16
**Plans:** 3/3 plans complete

Plans:

- [x] 17-01: Test isolation fixture + close_tab function
- [x] 17-02: Remove auto-sync, tab-level close on delete/archive, remove ArchiveModal
- [x] 17-03: Gap closure: focus new tab after h-key creation

**Details:**

Autouse session-scoped pytest fixture patches all five ~/.joy/ path constants (JOY_DIR, PROJECTS_PATH, CONFIG_PATH, REPOS_PATH, ARCHIVE_PATH) to a tmp directory — no test can touch real user data. Complemented by close_tab(tab_id) added to terminal_sessions.py. Auto-sync tab creation removed from refresh/new-project cycles; tabs only created via explicit h-key with in-flight guard. Delete and archive both close the entire iTerm2 tab before removing the project. ArchiveModal removed; replaced by ConfirmationModal with custom hint text. Newly created tabs are immediately focused via async_select() + async_activate().

## Milestone Summary

**Key Decisions:**

- Tab creation moved from auto-sync to explicit h-key press — user controls when tabs are created
- Always close tab on archive (no choice offered) — simplifies UX; ArchiveModal removed
- Session-scoped fixture (not function-scoped) for test isolation to avoid per-test overhead
- Virtual rows (REPO, TERMINALS, resolver worktrees) assemble in ProjectDetail alongside stored ObjectItems
- Per-kind DISPATCH table in dispatch.py replaces scattered if/else in app.py for keystroke routing
- clear_selection() (cursor=-1) replaces dimmed-state concept — unlinked items fully openable
- ArchivedProject wraps Project + archived_at; archive.toml uses keyed schema

**Issues Resolved:**

- Auto-sync creating unwanted iTerm2 tabs on every refresh/new-project
- Tests touching real ~/.joy/ causing cross-test contamination
- Tab not focused after h-key creation
- Source pane clearing its own selection during cross-pane sync
- Sync_to() not clearing selection on no-match in WorktreePane and TerminalPane

**Issues Deferred:**

- Pre-existing test failures in test_propagation.py (JoyApp._propagate_terminal_auto_remove) and test_sync.py (terminal sync) — out of scope for v1.3
- SYNC-10: Sync toggle state persisting across restarts — ephemeral is fine
- PERF-01: Real-time file watching on ~/.joy/ TOML — 30s refresh sufficient

**Technical Debt Incurred:**

- test_propagation.py::TestTerminalAutoRemove references non-existent method — needs cleanup in next milestone
