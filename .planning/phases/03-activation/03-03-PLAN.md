```python
def action_open_all_defaults(self) -> None:
    """Open all open_by_default objects for the current project (ACT-02, D-10)."""
    detail = self.query_one(ProjectDetail)
    project = detail._project
    if project is None:
        return  # silent no-op: data not loaded yet (D-11)
    # Collect defaults in GROUP_ORDER display order (D-06)
    defaults: list[ObjectItem] = []
    for kind in GROUP_ORDER:
        for item in project.objects:
            if item.kind == kind and item.open_by_default:
                defaults.append(item)
    if not defaults:
        return  # silent no-op: no defaults (D-11)
    self._open_defaults(defaults)
```

5. Add `_open_defaults` worker to `JoyApp`:
```python
@work(thread=True, exit_on_error=False)
def _open_defaults(self, defaults: list[ObjectItem]) -> None:
    """Open default objects sequentially in a background thread (D-06, D-07, D-08)."""
    from joy.operations import open_object  # noqa: PLC0415
    errors: list[str] = []
    for item in defaults:
        try:
            open_object(item=item, config=self._config)
            self.app.notify(
                _success_message(item, self._config),
                markup=False,
            )
        except Exception:
            display = _truncate(item.label if item.label else item.value)
            errors.append(display)
    # Show accumulated error toasts at the end (D-07)
    for err in errors:
        self.app.notify(f"Failed to open: {err}", severity="error", markup=False)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Bulk open -> multiple subprocesses | O fires multiple open_object calls sequentially in one worker |
| Object values -> toast | Same as Plan 02 -- user values in notification text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Denial of Service | _open_defaults worker | mitigate | `exit_on_error=False` prevents any single failure from crashing TUI. Continue-on-error loop ensures remaining objects still open (D-07). |
| T-03-03-02 | Information Disclosure | _open_defaults toasts | mitigate | All `app.notify()` calls use `markup=False` to prevent Rich markup injection from user values. |
| T-03-03-03 | Denial of Service | Bulk subprocess spawning | accept | Sequential execution (not concurrent) limits resource pressure. Maximum objects per project is bounded by user's own configuration (~10-20 typical). No external amplification vector. |
| T-03-03-04 | Elevation of Privilege | open_object subprocess calls | accept | Already mitigated in Phase 1: subprocess.run uses array form (no shell=True), AppleScript injection prevented by escaping. No new attack surface in bulk open. |
</threat_model>
