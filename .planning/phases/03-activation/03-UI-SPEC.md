# Phase 3 — UI Design Contract

> Visual and interaction contract for the Activation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 
> **Platform note:** joy is a Python TUI built with Textual 8.x. Design tokens map to Textual CSS variables, not web CSS. All spacing values are expressed in terminal cell units (1 cell = 1 character width or height), not pixels. Colors reference Textual's built-in theme system.

## Spacing Scale

Textual CSS uses integer cell units. All spacing in this contract is expressed in **terminal cells** (characters), not pixels. The 8-point grid does not apply to terminal UIs; instead, spacing uses cell-count conventions.

| Token | Value | Usage |
|-------|-------|-------|
| none | 0 | No gap between dot indicator and icon |
| xs | 1 | Inline padding (existing `padding: 0 1` on ObjectRow) |
| sm | 2 | Gap between icon and label, label and value (existing pattern) |
| md | 3 | Not used in this phase |
| lg | 4 | Not used in this phase |

Exceptions: none. Phase 3 adds only the dot indicator character to the existing row layout. No new spacing tokens introduced.

**Source:** Existing ObjectRow `_render_text` uses 2-space gaps between icon, label, and value.

## Typography

Textual renders in the terminal's monospace font. There are no font-size or font-weight choices in the traditional sense. Textual CSS supports `text-style` (bold, italic, underline) and `color` for emphasis.

| Role | Style | Usage |
|------|-------|-------|
| Body | normal | Object row values, project names |
| Label | normal | Preset kind labels in object rows (e.g., "branch", "mr") |
| Group header | bold + `$text-muted` color | Section headers ("Merge Requests", "Branches") -- existing Phase 2 pattern |
| Toast message | normal (Textual notify default) | Activation feedback messages |

No additional typography changes in Phase 3. All text rendering follows the existing Phase 2 patterns.

**Source:** Phase 2 codebase -- `GroupHeader` uses `text-style: bold; color: $text-muted;`, ObjectRow uses default text style.

## Color

Textual uses a built-in theme system with CSS variables. Phase 2 decision D-11 locked the app to Textual's default dark theme with no custom CSS overrides.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `$surface` (Textual default dark bg) | App background, pane backgrounds |
| Secondary (30%) | `$accent` (Textual default) | Highlighted/selected row background (existing), cursor row |
| Accent (10%) | `$text` (white/light in dark theme) | Filled default-open indicator dot (`U+25CF` = `●`) |
| Muted | `$text-muted` (Textual default) | Empty default-open indicator dot (`U+25CB` = `○`), group headers |
| Error | `severity="error"` on `app.notify()` | Failed activation toasts (red background, Textual built-in) |

**Accent reserved for:**
- Filled open-by-default indicator dot (`●`) on object rows where `open_by_default=True`
- Highlighted row background (existing Phase 2 behavior)

**Source:** 03-CONTEXT.md D-02 ("accent color or white when filled, muted color when empty"), Phase 2 D-11 (default dark theme).

## Object Row Layout Contract

Phase 3 modifies the ObjectRow rendering. This is the primary visual change.

### Current layout (Phase 2)

```
{icon}  {label}  {value}
```

Example: `  branch  feature/auth-refactor`

### New layout (Phase 3)

```
{dot} {icon}  {label}  {value}
```

Example (open-by-default ON):  `●   branch  feature/auth-refactor`
Example (open-by-default OFF): `○   branch  feature/auth-refactor`

### Specification

| Element | Character | Color when ON | Color when OFF |
|---------|-----------|---------------|----------------|
| Dot indicator | `●` (U+25CF) filled / `○` (U+25CB) empty | `$text` (white) | `$text-muted` (dim) |
| Gap after dot | 1 space | -- | -- |
| Icon | Nerd Font glyph (unchanged) | unchanged | unchanged |
| Gap after icon | 2 spaces (unchanged) | -- | -- |
| Label | preset kind name (unchanged) | unchanged | unchanged |
| Gap after label | 2 spaces (unchanged) | -- | -- |
| Value | object value or label, right-truncated with `...` (unchanged) | unchanged | unchanged |

### Implementation detail

The `ObjectRow._render_text()` method builds a `rich.text.Text` object. The dot character must be styled independently of the rest of the text using `Text.append()` with per-span styling.

**Source:** 03-CONTEXT.md D-01, D-02

## Interaction Contract

### Key Bindings

| Key | Scope | Action | Condition |
|-----|-------|--------|-----------|
| `o` | `ProjectDetail` (detail pane focused) | Open the highlighted object via `operations.open_object()` | Object must be highlighted; if none, show error toast |
| `O` | `JoyApp` (global -- any pane) | Open all `open_by_default=True` objects for the current project, sequentially in `GROUP_ORDER` | Uses currently highlighted project; silent no-op if no defaults exist |
| `space` | `ProjectDetail` (detail pane focused) | Toggle `open_by_default` on the highlighted object, persist to TOML, update dot indicator | Object must be highlighted |

**Source:** 03-CONTEXT.md D-09, D-10, D-11, D-12

### Activation Flow (single object -- `o`)

1. User presses `o` in the detail pane
2. `ProjectDetail.action_open_object()` reads `self.highlighted_object`
3. If `None`: show error toast "No object selected" via `app.notify("No object selected", severity="error")`
4. If valid: run `operations.open_object(item=item, config=config)` in a `@work(thread=True)` worker
5. On success: show toast with type-specific message (see Copywriting Contract below)
6. On `subprocess.CalledProcessError` or other exception: show error toast with message

### Bulk Activation Flow (`O`)

1. User presses `O` from any pane
2. `JoyApp.action_open_all_defaults()` collects objects where `open_by_default=True` from the current project
3. If no default objects: silent no-op (no toast, no error)
4. Iterate sequentially in `GROUP_ORDER` display order
5. For each object: call `operations.open_object()` in the same background thread
6. On success of each: show one toast per object (same format as single `o`)
7. On failure of any: continue to next object; show one error toast per failure at end

**Source:** 03-CONTEXT.md D-06, D-07, D-08

### Toggle Flow (`space`)

1. User presses `space` in the detail pane
2. `ProjectDetail.action_toggle_default()` reads `self.highlighted_object`
3. Flip `item.open_by_default` boolean
4. Update the `ObjectRow` dot indicator in-place (change character and color)
5. Persist via `store.save_projects()` in a `@work(thread=True)` worker
6. No toast feedback for toggle (visual dot change is sufficient immediate feedback)

**Source:** 03-CONTEXT.md D-12

## Copywriting Contract

### Toast Messages (activation feedback)

| Object Type | Success Message Format | Example |
|-------------|----------------------|---------|
| `string` (clipboard) | `Copied: {value_truncated}` | `Copied: feature/auth-refactor` |
| `url` (browser) | `Opened: {value_truncated}` | `Opened: gitlab.com/user/joy/-/merge_...` |
| `url` (Notion) | `Opened in Notion: {value_truncated}` | `Opened in Notion: notion.so/ticket-1...` |
| `url` (Slack) | `Opened in Slack: {value_truncated}` | `Opened in Slack: app.slack.com/clien...` |
| `obsidian` | `Opened in Obsidian: {value_truncated}` | `Opened in Obsidian: Projects/joy` |
| `file` | `Opened in {editor}: {value_truncated}` | `Opened in Sublime Text: src/main.py` |
| `worktree` | `Opened in {ide}: {value_truncated}` | `Opened in PyCharm: /Users/pieter/Git...` |
| `iterm` | `Opened in iTerm2: {value_truncated}` | `Opened in iTerm2: joy-agents` |

**Value truncation rule:** If value exceeds 40 characters, truncate with `...` at character 37 + `...`. Display the label if `item.label` is non-empty; otherwise display `item.value`.

**Source:** 03-CONTEXT.md D-04

### Error Messages

| Condition | Message | Severity |
|-----------|---------|----------|
| No object highlighted when `o` pressed | `No object selected` | `error` |
| Subprocess failure on single open | `Failed to open: {value_truncated}` | `error` |
| Subprocess failure during bulk open | `Failed to open: {value_truncated}` (one per failure, shown after all attempts) | `error` |

**Source:** 03-CONTEXT.md D-05, D-07, D-11

### Empty States

| Condition | Behavior |
|-----------|----------|
| No default objects when `O` pressed | Silent no-op -- no toast, no error |
| Empty project (no objects) when detail pane focused | `o` shows error toast "No object selected" |

### Destructive Actions

None in Phase 3. Toggle (`space`) is non-destructive and reversible. No confirmation dialogs needed.

## Toast Notification Specification

Using Textual's built-in `app.notify()` method.

| Property | Value |
|----------|-------|
| Method | `self.app.notify(message, severity=severity)` |
| Default severity | `"information"` (for successful activations) |
| Error severity | `"error"` (for failures) |
| Duration | Textual default (auto-dismiss) |
| Position | Textual default (bottom-right floating toast) |

No custom notification widget needed.
