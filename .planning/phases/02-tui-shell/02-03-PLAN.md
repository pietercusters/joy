  <action>
Textual's built-in `Footer` widget automatically reads BINDINGS from the focused widget up through ancestors and renders them. This already provides context-sensitive binding display (D-07). However, D-08 requires a pane context label prefix (e.g., "Projects |" or "Detail |").

Approach: Use Textual's built-in Footer (which already renders bindings context-sensitively). For the pane label (D-08), watch focus changes and update the app's sub_title or a custom header to indicate which pane is active. Since Textual Footer does not natively support a prefix label, we have two options:

**Option A (simpler, recommended):** Use Textual's `Header` widget to show the pane context. The Header displays `TITLE` and `SUB_TITLE`. Set `SUB_TITLE` reactively based on which pane has focus. The Footer shows bindings only. This keeps the pane context label at the top (visible and clear) and bindings at the bottom.

**Option B:** Subclass Footer to prepend the pane label. This is more complex and fragile.

Implement Option A:

1. In `JoyApp`, add `from textual.widgets import Header, Footer`.

2. In `compose()`, yield `Header()` before the Horizontal, and `Footer()` after. The Header will show "joy" (TITLE) and the pane context as SUB_TITLE.

3. Add a reactive `SUB_TITLE` that updates based on focus:
   ```python
   SUB_TITLE = "Projects"
   ```

4. Override `watch_focused(self, focused)` or use `on_descendant_focus` to detect which pane has focus. When the focused widget is inside `#project-list` (the ListView), set `self.sub_title = "Projects"`. When focused widget is `#project-detail`, set `self.sub_title = "Detail"`.

   Implementation:
   ```python
   def on_descendant_focus(self, event) -> None:
       """Update sub_title based on which pane has focus (D-08)."""
       focused = event.widget
       # Walk up the DOM to find if we're in project-list or project-detail
       widget = focused
       while widget is not None:
           if widget.id == "project-list" or (hasattr(widget, 'id') and widget.id == "project-listview"):
               self.sub_title = "Projects"
               return
           if widget.id == "project-detail":
               self.sub_title = "Detail"
               return
           widget = widget.parent
   ```

   Actually, a simpler approach: use `on_focus` handler on the app. Or check `self.focused`:
   ```python
   def watch_focused(self, focused: Widget | None) -> None:
       if focused is None:
           return
       # Check if the focused widget or any ancestor has the detail ID
       node = focused
       while node is not None:
           if hasattr(node, 'id'):
               if node.id == "project-detail":
                   self.sub_title = "Detail"
                   return
               if node.id in ("project-list", "project-listview"):
                   self.sub_title = "Projects"
                   return
           node = node.parent
   ```

   Note: `focused` is a reactive on App so we can use `watch_focused`.

5. The Footer bindings update automatically when focus changes between panes. When focus is on `#project-listview` (ListView inside ProjectList), Footer shows ListView's built-in bindings (up/down/enter) plus JoyApp's `q`. When focus is on `#project-detail` (ProjectDetail), Footer shows its BINDINGS (j/k/up/down/escape) plus JoyApp's `q`.

6. Add `("q", "quit", "Quit")` at the JoyApp level (already present from Plan 01). This ensures q is always shown regardless of pane focus.

7. Verify the footer shows bindings by reviewing what BINDINGS each widget exposes:
   - JoyApp: `q` (Quit)
   - ProjectDetail: `escape` (Back), `j`/`k`/`up`/`down` (navigation)
   - ListView (built-in): has its own internal bindings for up/down/enter

   The Footer widget renders all bindings from the focus chain. This satisfies D-07 (all available bindings shown).

```python
"""Textual pilot tests for the joy TUI."""
from __future__ import annotations

import pytest
from unittest.mock import patch
from joy.app import JoyApp
from joy.models import Project, ObjectItem, PresetKind


def _sample_projects() -> list[Project]:
    """Create sample projects for testing."""
    return [
        Project(
            name="project-alpha",
            objects=[
                ObjectItem(kind=PresetKind.BRANCH, value="main"),
                ObjectItem(kind=PresetKind.MR, value="https://example.com/mr/1", label="MR #1"),
                ObjectItem(kind=PresetKind.WORKTREE, value="/tmp/alpha"),
            ],
        ),
        Project(
            name="project-beta",
            objects=[
                ObjectItem(kind=PresetKind.TICKET, value="https://notion.so/ticket-1", label="TICK-1"),
            ],
        ),
    ]


@pytest.fixture
def mock_store():
    """Mock store.load_projects to return sample data without touching ~/.joy/."""
    with patch("joy.store.load_projects", return_value=_sample_projects()) as mock:
        yield mock


@pytest.mark.asyncio
async def test_app_launches_with_two_panes(mock_store):
    """CORE-01: App shows two-pane layout."""
    app = JoyApp()
    async with app.run_test() as pilot:
        # Both panes should be present in the DOM
        assert app.query_one("#project-list") is not None
        assert app.query_one("#project-detail") is not None


@pytest.mark.asyncio
async def test_first_project_auto_selected(mock_store):
    """PROJ-02: First project is auto-selected on startup."""
    app = JoyApp()
    async with app.run_test() as pilot:
        await pilot.pause()  # Let workers complete
        # The detail pane should show the first project's data
        detail = app.query_one("#project-detail")
        assert detail._project is not None
        assert detail._project.name == "project-alpha"


@pytest.mark.asyncio
async def test_enter_shifts_focus_to_detail(mock_store):
    """D-04: Enter on project shifts focus to detail pane."""
    app = JoyApp()
    async with app.run_test() as pilot:
        await pilot.pause()
        # Focus should start on the project list (ListView)
        await pilot.press("enter")
        await pilot.pause()
        # After Enter, focus should be on the detail pane
        focused = app.focused
        assert focused is not None
        assert focused.id == "project-detail"


@pytest.mark.asyncio
async def test_escape_returns_focus_to_list(mock_store):
    """D-06, CORE-04: Escape returns focus from detail to project list."""
    app = JoyApp()
    async with app.run_test() as pilot:
        await pilot.pause()
        # Move to detail pane
        await pilot.press("enter")
        await pilot.pause()
        # Press escape to go back
        await pilot.press("escape")
        await pilot.pause()
        # Focus should be back on the list
        focused = app.focused
        assert focused is not None
        # Should be the listview inside project-list
        assert focused.id in ("project-listview", "project-list")


@pytest.mark.asyncio
async def test_quit_with_q(mock_store):
    """App quits when q is pressed."""
    app = JoyApp()
    async with app.run_test() as pilot:
        await pilot.press("q")
        # App should have exited (no assertion needed -- if it hangs, test fails by timeout)
```

Note: The mock patches `joy.store.load_projects` so the worker thread in JoyApp._load_data picks up the mock. If the lazy import pattern (importing inside the worker) doesn't pick up the mock, patch at the module where it's called: `patch("joy.app.load_projects")` -- but since the import happens inside the function, we need to patch `joy.store.load_projects` before the function runs. The `mock_store` fixture patches it before the app starts, so the lazy import will find the patched version.

If the above approach fails (because of the lazy import inside `_load_data`), refactor `_load_data` in app.py to call a module-level function that can be patched, or use `patch.dict("sys.modules", ...)`. The simplest fix is to patch at the store module level: `patch("joy.store.load_projects")`.
