**Module docstring and imports:**
```python
"""Git worktree discovery for registered repos. No UI, no persistence."""
from __future__ import annotations

import subprocess
from joy.models import Repo, WorktreeInfo
```

**Public API — single function:**
```python
def discover_worktrees(
    repos: list[Repo],
    branch_filter: list[str] | None = None,
) -> list[WorktreeInfo]:
```

Parameters:
- `repos`: List of Repo objects (from `load_repos()`)
- `branch_filter`: List of exact branch names to EXCLUDE from results. `None` or `[]` means no filtering. Per D-01, uses exact string match (`branch == filter_value`), NOT fnmatch/glob.

Returns: `list[WorktreeInfo]` — all active worktrees across all valid repos, minus filtered branches. Per D-02, repos with missing paths or git errors are silently skipped.

**Implementation structure (3 internal helper functions + 1 public function):**

1. `_list_worktrees(repo_path: str) -> list[tuple[str, str]]`
   - Runs `git -C {repo_path} worktree list --porcelain`
   - Parses porcelain output format. Each worktree block is separated by a blank line and contains lines like:
     ```
     worktree /path/to/worktree
     HEAD abc123def
     branch refs/heads/main
     ```
     For detached HEAD, the line is `detached` instead of `branch refs/heads/...`.
   - Returns list of `(path, branch)` tuples. Branch is the short name (strip `refs/heads/` prefix). For detached HEAD, use `"HEAD"`.
   - On any subprocess error (non-zero exit, timeout, OSError): return `[]`

2. `_is_dirty(worktree_path: str) -> bool`
   - Runs `git -C {worktree_path} diff-index --quiet HEAD --`
   - Exit code 0 = clean (return False), exit code 1 = dirty (return True)
   - On any subprocess error: return False (assume clean — conservative default)
   - Note: `diff-index` only checks tracked files. Untracked files are NOT flagged.

3. `_has_upstream(worktree_path: str) -> bool`
   - Runs `git -C {worktree_path} rev-parse --abbrev-ref --symbolic-full-name @{u}`
   - Exit code 0 and non-empty stdout = has upstream (return True)
   - Exit code 128 (no upstream configured) = no upstream (return False)
   - On any subprocess error: return False (assume no upstream — conservative)

4. `discover_worktrees(repos, branch_filter)` — the public function:
   ```
   filter_set = set(branch_filter) if branch_filter else set()
   results = []
   for repo in repos:
       worktrees = _list_worktrees(repo.local_path)
       # _list_worktrees returns [] on any error, so invalid repos are silently skipped (D-02)
       for path, branch in worktrees:
           if branch in filter_set:  # exact match per D-01
               continue
           results.append(WorktreeInfo(
               repo_name=repo.name,
               branch=branch,
               path=path,
               is_dirty=_is_dirty(path),
               has_upstream=_has_upstream(path),
           ))
   return results
   ```

**Subprocess pattern (match store.py exactly):**
- Always use list-form: `["git", "-C", path, ...]` — NEVER `shell=True`
- Always use `capture_output=True, text=True, timeout=5`
- Always use `check=False` — inspect `result.returncode` manually
- Wrap in `try/except (subprocess.TimeoutExpired, OSError)` — return safe default on failure

**Porcelain parsing detail for `_list_worktrees`:**
The output of `git worktree list --porcelain` looks like:
```
worktree /Users/dev/project
HEAD abc123
branch refs/heads/main

worktree /Users/dev/project-feature
HEAD def456
branch refs/heads/feature/x

```
Split on double-newline (`\n\n`) to get blocks. For each block, extract the `worktree` line (path) and `branch` line (strip `refs/heads/`). If no `branch` line exists but a `detached` line does, use `"HEAD"`. Skip any block that has a `bare` line (bare worktrees are the .git directory itself in non-standard setups — but note: `git worktree list --porcelain` marks the main worktree differently, it does NOT have a `bare` line unless the repo itself is bare).
