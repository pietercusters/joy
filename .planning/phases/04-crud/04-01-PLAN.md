**src/joy/screens/preset_picker.py — PresetPickerModal(ModalScreen[PresetKind | None]):**
- Imports: PresetKind from joy.models, PRESET_ICONS from joy.widgets.object_row, GROUP_ORDER from joy.widgets.project_detail, Input/ListView/ListItem/Label/Static from textual.widgets, Vertical from textual.containers, ModalScreen from textual.screen, ComposeResult from textual.app, Key from textual.events
- Class: `PresetPickerModal(ModalScreen[PresetKind | None])`
- `BINDINGS = [("escape", "cancel", "Cancel")]`
- Class attribute `ALL_PRESETS: list[PresetKind] = list(GROUP_ORDER)` (uses GROUP_ORDER for consistent display order)
- `DEFAULT_CSS`: align center middle; inner Vertical: `width: 60; height: auto; max-height: 20; background: $surface; border: thick $background 80%; padding: 1 2;`
- Also: `#preset-list { height: auto; max-height: 12; }` (so 9 items fit without scroll, but clips if terminal tiny)
- Instance variable `self._filtered: list[PresetKind]` initialized to `list(self.ALL_PRESETS)` in `__init__`
- `compose()`: yield Vertical containing `Static("Add Object", classes="modal-title")`, `Input(placeholder="Type to filter...", id="filter-input")`, `ListView(*[ListItem(Label(f"{PRESET_ICONS[k]}  {k.value}")) for k in self.ALL_PRESETS], id="preset-list")`, `Static("j/k to navigate, Enter to select, Escape to cancel", classes="modal-hint")`
- `on_mount()`: `self.query_one("#filter-input", Input).focus()`
- `on_input_changed(event: Input.Changed)`: filter ALL_PRESETS by `event.value.lower()` substring match on `k.value`; rebuild ListView via `listview.clear()` + loop `listview.append(ListItem(Label(f"{PRESET_ICONS[kind]}  {kind.value}")))` for each match; update `self._filtered`
- `on_key(event: Key)`: Intercept j/k while Input has focus to navigate ListView. If `event.key == "j"` and filter-input is focused: `event.prevent_default(); event.stop(); listview.action_cursor_down()`. Same for `"k"` with `action_cursor_up()`. This prevents j/k being typed as text and instead navigates the list (per Research open question 1 recommendation).
- `on_list_view_selected(event: ListView.Selected)`: if `self._filtered` not empty and `event.list_view.index` is valid: `self.dismiss(self._filtered[event.list_view.index])`
- `on_input_submitted(event: Input.Submitted)`: When Enter pressed in filter input: if `self._filtered` has exactly 1 item, dismiss with it. If multiple items, move focus to ListView and highlight first item (or select first from self._filtered based on listview.index).
- `action_cancel()`: `self.dismiss(None)`

The test pattern:
```python
from textual.app import App, ComposeResult
from textual.widgets import Static

class ModalTestApp(App):
    """Minimal app for testing modal screens."""
    def compose(self) -> ComposeResult:
        yield Static("test")
```

For each modal test:
1. Create ModalTestApp instance
2. Use `async with app.run_test() as pilot`
3. Push the modal with a callback that captures the result
4. Interact with pilot.press() to type/navigate
5. Assert the captured result

Key implementation details:
- Use `result_holder = []` pattern to capture callback results (closures)
- Use `await pilot.pause(0.1)` after push_screen to let modal mount
- For typing text: `for ch in "text": await pilot.press(ch)` (not `pilot.press(*"text")` which unpacks differently)
- For PresetPickerModal filter test: type "br", then read ListView children count — should be 1 with label containing "branch"
- For ConfirmationModal: after push, the screen itself needs focus for Enter/Escape to work — test that `pilot.press("enter")` and `pilot.press("escape")` produce correct results
- Import PresetKind from joy.models for PresetPickerModal and ValueInputModal tests

All tests should be `@pytest.mark.asyncio async def test_...` following existing convention.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4-01 | Spoofing | NameInputModal | accept | Single-user local tool, no authentication needed. User is the only actor. |
| T-4-02 | Tampering | Input widgets | mitigate | All user input is stripped (`.strip()`) before use. Values are serialized by `tomli_w` which escapes TOML special characters. No code injection path. |
| T-4-03 | Information Disclosure | ConfirmationModal | accept | Modal shows project name and object value to the same user who created them. No disclosure risk. |
| T-4-04 | Denial of Service | PresetPickerModal | accept | ListView rebuilds on every keystroke for a fixed set of 9 items. No performance concern. |
| T-4-05 | Injection | notify() calls | mitigate | All `app.notify()` calls use `markup=False` per project-wide convention (Phase 3 decision). User-supplied strings are never interpreted as Rich markup. |
