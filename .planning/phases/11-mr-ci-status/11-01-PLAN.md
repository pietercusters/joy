Create src/joy/mr_status.py with these functions:

1. `fetch_mr_data(repos: list[Repo], worktrees: list[WorktreeInfo]) -> dict[tuple[str, str], MRInfo]`
   - Build `branches_by_repo: dict[str, set[str]]` from worktrees (key=repo_name, value=set of branch names)
   - Iterate repos: skip if `repo.forge == "unknown"` (D-07)
   - For each repo: try/except wrapping the forge-specific fetch (D-11)
   - GitHub repos: call `_fetch_github_mrs(repo, active_branches)`
   - GitLab repos: call `_fetch_gitlab_mrs(repo, active_branches)`
   - On any exception per repo: `continue` (silent skip, partial results returned)
   - Return accumulated dict. Module never raises.

2. `_fetch_github_mrs(repo: Repo, active_branches: set[str]) -> dict[tuple[str, str], MRInfo]`
   - Run: `subprocess.run(["gh", "pr", "list", "-R", repo.remote_url, "--json", "number,headRefName,isDraft,author,commits,statusCheckRollup", "--state", "open"], capture_output=True, text=True, check=False, timeout=15)`
   - If returncode != 0: raise RuntimeError(result.stderr.strip())
   - Parse JSON. For each PR where `headRefName` is in `active_branches`:
     - Extract `commits[-1]` for last commit (oid[:7], messageHeadline). Handle empty commits list.
     - Map `statusCheckRollup` through `_map_gh_ci_status()`
     - Create MRInfo with `author=f"@{pr['author']['login']}"`
   - Return dict keyed by `(repo.name, branch)`

3. `_fetch_gitlab_mrs(repo: Repo, active_branches: set[str]) -> dict[tuple[str, str], MRInfo]`
   - Run: `subprocess.run(["glab", "mr", "list", "-R", repo.remote_url, "--output", "json", "--per-page", "100"], capture_output=True, text=True, check=False, timeout=15)`
   - If returncode != 0: raise RuntimeError(result.stderr.strip())
   - Parse JSON. For each MR where `source_branch` is in `active_branches`:
     - Fetch CI status via `_fetch_glab_ci_status(repo, branch)`
     - Create MRInfo with `author=f"@{mr['author']['username']}"`, `last_commit_hash=""`, `last_commit_msg=""` (not available from list endpoint per RESEARCH.md)
   - Return dict keyed by `(repo.name, branch)`

4. `_fetch_glab_ci_status(repo: Repo, branch: str) -> str | None`
   - Run: `subprocess.run(["glab", "ci", "get", "-R", repo.remote_url, "--branch", branch, "--output", "json"], capture_output=True, text=True, check=False, timeout=15)`
   - If returncode != 0: return None
   - Parse JSON, extract `status` field, map through `_map_glab_ci_status()`
   - On JSONDecodeError: return None

5. `_map_gh_ci_status(rollup: list[dict]) -> str | None`
   - Empty rollup: return None
   - Any check with `status != "COMPLETED"`: return "pending"
   - Any check with `conclusion in {"FAILURE", "TIMED_OUT", "ACTION_REQUIRED", "CANCELLED"}`: return "fail"
   - Otherwise: return "pass"

6. `_map_glab_ci_status(status: str | None) -> str | None`
   - `status in ("running", "pending", "created", "preparing", "waiting_for_resource", "scheduled")`: return "pending"
   - `status == "success"`: return "pass"
   - `status == "failed"`: return "fail"
   - Otherwise (including None, "canceled", "skipped", "manual"): return None

| Boundary | Description |
|----------|-------------|
| joy -> gh CLI | Subprocess call to external CLI; output is untrusted JSON |
| joy -> glab CLI | Subprocess call to external CLI; output is untrusted JSON |
| gh/glab -> remote API | CLI handles auth/TLS; joy trusts the CLI's auth model |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | T (Tampering) | gh/glab JSON output | accept | CLI output is from authenticated API; no additional verification needed for a personal tool |
| T-11-02 | D (Denial of Service) | subprocess.run hangs | mitigate | timeout=15 on all subprocess.run calls in mr_status.py |
| T-11-03 | I (Info Disclosure) | stderr from gh/glab | mitigate | stderr is used for error detection internally, never displayed to user (D-10 shows generic message) |
