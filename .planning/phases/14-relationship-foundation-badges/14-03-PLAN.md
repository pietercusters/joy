  ```python
  def _maybe_compute_relationships(self) -> None:
      """Compute RelationshipIndex when both workers have completed their cycle (D-07, D-08).

      Called from _set_worktrees and _set_terminal_sessions — both run on the main thread
      via call_from_thread, so no asyncio coordination needed. Uses two boolean flags.
      Ready-flags are reset immediately to prevent stale-data races on subsequent cycles
      (see RESEARCH.md Pitfall 1).
      """
      if not (self._worktrees_ready and self._sessions_ready):
          return
      # Reset flags before computing (D-07: prevents stale-data on next cycle)
      self._worktrees_ready = False
      self._sessions_ready = False
      from joy.resolver import compute_relationships  # noqa: PLC0415 — lazy import per project convention
      self._rel_index = compute_relationships(
          self._projects,
          self._current_worktrees,
          self._current_sessions,
          self._repos,
      )
      self._update_badges()

  def _update_badges(self) -> None:
      """Push RelationshipIndex badge counts to ProjectList rows (D-08, D-11, BADGE-03)."""
      if self._rel_index is None:
          return
      from joy.widgets.project_list import ProjectList  # noqa: PLC0415
      try:
          self.query_one(ProjectList).update_badges(self._rel_index)
      except Exception:
          pass  # ProjectList not yet mounted — badges will be populated on next cycle
  ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| background thread → main thread | _set_worktrees and _set_terminal_sessions are called via call_from_thread; _maybe_compute_relationships runs on main thread synchronously |
| resolver → ProjectList | compute_relationships() result is passed to update_badges(); resolver is pure so no side-effects leak |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-07 | Tampering | _current_worktrees / _current_sessions on JoyApp | accept | Internal app state; only modified by _set_worktrees and _set_terminal_sessions on main thread; no external write path |
| T-14-08 | Denial of Service | _maybe_compute_relationships called rapidly | accept | Called at most twice per refresh cycle (once per worker callback); resolver is O(n); no unbounded loop risk |
| T-14-09 | Reliability | Ready-flags not reset → stale badge counts on 2nd refresh | mitigate | Flags reset to False inside _maybe_compute_relationships() before computing (RESEARCH.md Pitfall 1 explicitly addressed) |
| T-14-10 | Reliability | sessions=None treated as [] for resolver | mitigate | `self._current_sessions = sessions or []` in _set_terminal_sessions handles iTerm2 unavailable case (RESEARCH.md Pitfall 2) |
| T-14-11 | Information Disclosure | ICON constants imported from widget modules | accept | Icon constants are Unicode codepoints for Nerd Font glyphs; no sensitive data; cross-widget import is one-way (project_list ← worktree_pane, terminal_pane) |
</threat_model>
