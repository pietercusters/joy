<objective>
Add identity-based cursor preservation to `WorktreePane.set_worktrees()` and `TerminalPane.set_sessions()`. Before a DOM rebuild triggered by background refresh, each pane saves the currently-highlighted item's identity. After rebuilding, it searches new rows for that identity and restores the cursor — preventing the jarring jump-to-row-0 that currently occurs on every refresh.

Purpose: This is a prerequisite for Phase 15 (cross-pane sync) — sync operations need to know which item was active before the rebuild, not always the first row.
Output: Modified pane files with cursor preservation. New slow-marked TUI tests proving the behavior.
</objective>
