<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Config fields | Free-text strings from settings modal Input widgets written to config.toml |
| Config fields -> TOML file | Config values serialized via tomli_w to ~/.joy/config.toml |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | T (Tampering) | SettingsModal field values | accept | Local single-user tool; config values are paths/app names used locally. tomli_w handles TOML escaping. No injection vector. |
| T-05-01-02 | I (Information Disclosure) | Config file at ~/.joy/config.toml | accept | No secrets stored. Config contains IDE name, editor name, vault path, terminal name. All non-sensitive. |
| T-05-01-03 | S (Spoofing) | SelectionList.selected type mismatch | mitigate | Pass `k.value` (str) as SelectionList value param, not PresetKind enum. Verified in RESEARCH.md. Config.default_open_kinds expects list[str]. |
| T-05-01-04 | D (Denial of Service) | save_config blocking main thread | mitigate | save_config runs in @work(thread=True) background thread. Same pattern as _save_projects_bg. |
| T-05-01-05 | T (Tampering) | Rich markup injection via notify() | mitigate | All notify() calls use markup=False per project convention. |
</threat_model>
